## Security Considerations

### OTP Expiry
- All OTPs expire after 10 minutes
- Expired OTPs are rejected with clear error message
- Users can request new OTP via resend endpoint

### One-Time Use
- OTPs are cleared after successful verification
- Cannot reuse the same OTP
- New OTP required for each verification

### Rate Limiting (Recommended)
```javascript
// Add rate limiting to prevent OTP spam
const rateLimit = require('express-rate-limit');

const otpLimiter = rateLimit({
  windowMs: 15 * 60 * 1000, // 15 minutes
  max: 5, // 5 requests per window
  message: 'Too many OTP requests. Please try again later.'
});

router.post('/resend-otp', otpLimiter, authController.resendOTP);
```

### Account Lockout (Recommended)
```javascript
// Lock account after 5 failed OTP attempts
const MAX_FAILED_ATTEMPTS = 5;
const LOCKOUT_DURATION = 30 * 60 * 1000; // 30 minutes

// Track failed attempts in database
ALTER TABLE users ADD COLUMN failed_otp_attempts INT DEFAULT 0;
ALTER TABLE users ADD COLUMN locked_until DATETIME;
```

## Error Handling

### Common Errors

| Error | Cause | Solution |
|-------|-------|----------|
| "User not found" | Invalid userId | Check userId from signup response |
| "OTP expired" | OTP older than 10 minutes | Request new OTP |
| "Invalid OTP" | Wrong OTP entered | Check email/console for correct OTP |
| "Email already verified" | Trying to verify again | Redirect to login |
| "Email not verified" | Login without verification | Complete signup verification first |
| "No OTP found" | OTP not generated | Request new OTP |

### Error Response Format
```json
{
  "status": "error",
  "message": "OTP has expired. Please request a new one.",
  "code": "OTP_EXPIRED"
}
```
